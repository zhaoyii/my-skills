# VSCode Agent 编程工作流优化方案

## 问题分析

### 1. 屏幕空间不足
- 左右分栏布局下，Claude Code Agent 面板空间狭小
- 即使使用大显示器，Agent 面板占比仍显不足

### 2. 多 Agent 场景拥挤
- 同时运行多个 Claude Code Agent 时，单个显示器难以容纳
- 窗口堆叠导致操作互相遮挡

### 3. 布局理念冲突
- **传统编程模式**：左侧文件树 + 中间代码编辑区 = 主位，Agent 面板 = 辅助
- **Agent 优先模式**：Agent 面板应为主位，文件树退居次位
- 现有 IDE 布局未适配 Agent 时代的编程范式

---

## 解决方案

### 核心操作步骤

1. **创建独立 Agent 窗口**
   - 使用 `Ctrl+Shift+P` 打开命令面板
   - 执行 "New Window" 创建新窗口
   - 每个 Agent 分配一个独立 VSCode 窗口

2. **固定 Agent 窗口**
   - 使用系统工具（如 top、PowerToys）将 Agent 窗口置顶
   - 确保 Agent 输出始终可见

3. **人工 Review 阶段**
   - 编程任务完成后，取消窗口置顶
   - 切换至文件树窗口进行人工检查
   - 确认无误后关闭 Agent 窗口

### 布局示意

```
+-------------------------------+
|  Agent 窗口 (top 置顶)        |
|  +-------------------------+  |
|  | Claude Code Agent 输出  |  |
|  |                         |  |
|  |                         |  |
|  +-------------------------+  |
+-------------------------------+
|  文件树窗口 (后台待命)         |
|  +-------------------------+  |
|  | 文件树 / 代码编辑区     |  |
|  |                         |  |
|  +-------------------------+  |
+-------------------------------+
```

---

## 核心理念

| 阶段 | 主位 | 次位 |
|------|------|------|
| 编程中 | Agent 面板 | 文件树 |
| Review 时 | 文件树 | Agent 面板 |

**核心原则**：Agent 优先，人工兜底

---

## 补充建议

- **多显示器**：将 Agent 窗口拖至副屏，文件树留在主屏
- **窗口管理**：使用 `Win+方向键` 或窗口管理插件优化布局
- **工作区分离**：不同项目的 Agent 窗口独立，避免干扰
