---
name: verification-loop
description: 强制 Claude 在「验证 → 观察 → 修正」的闭环中工作。适用于编写代码、修改行为或修复 Bug 等对正确性要求高于速度的场景。
---

当启用该 skill 时，始终遵循以下流程：

1. **先想验证，再写代码**

   在编写或修改任何代码之前，必须先回答：
   - 什么可观察的结果可以证明它是有效的？
   - 我如何亲自观察到这个结果（通过工具、输出、UI 或命令）？

   在给出明确的验证方式之前，不得开始写代码。

2. **显式声明验证方式**

   必须按如下格式清晰说明验证计划：
   - 验证方式： （命令 / 测试 / UI 操作 / 输出）
   - 期望看到的结果： （具体、可观察的结果）

   如果结果无法被直接观察，则说明任务尚未准备好。

3. **实现最小可验证改动**
   - 只编写触发验证所需的最少代码
   - 此阶段避免重构、清理或优化

   目标是尽快进入验证，而不是一次性完成方案。

4. **执行验证并观察结果**

   在实现之后：
   - 执行已声明的验证方式
   - 观察实际发生的结果
   - 将实际结果与期望结果进行对比

   必须明确说明验证是**通过**还是**失败**。

5. **验证失败则修正并重复**

   当验证失败时：
   - 描述你实际观察到的现象，而不是原本的预期
   - 只修改与失败直接相关的部分
   - 重新执行同一验证方式

   持续循环，直到验证通过。

6. **仅在验证通过后才能继续**

   只有在验证成功之后，才允许：
   - 清理代码
   - 改进结构
   - 添加测试
   - 准备提交或 PR

**经验法则**

如果我无法亲自验证它，那么它就还没有完成。
