---
name: data-structure-first
description: 优先设计数据结构，再设计算法。当算法或者程序复杂时，优化数据结构。适用于任何需要解决复杂问题的编程任务。
---

当设计程序，编写代码时，始终遵循：

1. **先设计数据结构，再设计算法**：将功能设计，转换成数据结构的设计。在编写任何算法代码之前，必须先明确数据的组织方式，数据之间的关系。
2. **优先使用 List 和 HashMap**：优先使用 List 和 HashMap 来存储数据，避免使用复杂的数据结构。在才用复杂数据结构时，必须有可靠的依据。
3. **用数据结构表达语义**：用枚举，而非布尔参数；用专门的结构体，而非散乱的数据集合；用有意义的名称表达业务概念；让数据结构直接对应问题领域的概念。
4. **警惕过早的算法优化**：如果一个算法实现过于复杂，停下来思考是否可以用更好的数据结构替代。不要用算法技巧掩盖数据结构设计的缺陷，不要在错误的数据结构上强行优化。
5. **数据驱动设计**：让数据结构驱动程序逻辑，而不是让程序逻辑驱动数据结构。数据结构应该反映业务需求和用例。
