---
name: data-structure-first
description: 强调优先设计数据结构，算法自然简化。适用于任何需要解决复杂问题的编程任务。
---

当启用该 skill 时，始终遵循以下原则：

1. **先设计数据结构，再考虑算法**

在编写任何算法代码之前，必须先明确数据的组织方式。
- 数据如何存储？
- 数据之间的关系是什么？
- 需要支持哪些操作？

2. **好的数据结构让算法变简单**

时刻谨记：选择正确的数据结构，算法往往自然简化。
- 如果一个算法实现过于复杂，停下来思考是否可以用更好的数据结构替代
- 复杂查询往往可以通过合理的结构设计变成简单查找
- 优先考虑 O(1) 的数据结构操作，而非 O(n) 的算法优化

3. **数据结构即注释**

代码应当自文档化。好的数据结构设计应当：
- 名称清晰表达意图
- 结构本身就能说明数据的含义
- 减少对注释的依赖

4. **先建模，后编码**

在动手写代码前，必须完成数据建模：
- 定义核心数据结构（结构体、类、表结构等）
- 明确数据间的关联关系
- 确定操作的接口（而不是具体实现）

5. **用数据结构表达语义**

- 用枚举而非布尔参数
- 用专门的结构体而非散乱的数据集合
- 用有意义的名称表达业务概念
- 让数据结构直接对应问题领域的概念

6. **警惕过早的算法优化**

在数据结构设计不当的情况下：
- 不要用算法技巧掩盖数据结构设计的缺陷
- 不要在错误的数据结构上强行优化
- 首先确保数据结构正确反映问题

**经验法则**

如果一个问题的算法实现很复杂，那很可能是数据结构设计不够好。重新审视数据结构，往往能找到更优雅的解决方案。
