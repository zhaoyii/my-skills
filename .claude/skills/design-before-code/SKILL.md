---
name: design-before-code
description: 要求 Claude 在写代码前先完成程序设计，明确问题、目标以及环境约束。适用于任何非平凡的工程任务。
---

When this skill is active, always follow this process:

1. **禁止直接写代码**
在设计完成之前，不得编写、修改或生成任何代码。

2. **明确问题本身**
在设计阶段，必须回答：
- 要解决的核心问题是什么？
- 为什么需要解决这个问题？
- 如果不解决，会带来什么后果？

3. **明确目标与成功标准**
设计中必须说明：
- 期望达到的最终结果是什么？
- 什么条件下可以认为任务已完成？
- 是否存在明确的成功或失败判定标准？

4. **明确环境与约束条件**
必须显式列出相关约束，包括但不限于：
- 运行环境与技术栈
- 现有系统或代码边界
- 性能、成本、安全或复杂度限制
- 明确禁止的做法

5. **给出设计方案**
设计应描述：
- 整体结构或模块划分
- 关键职责边界
- 主要数据流或控制流程
- 重要设计取舍及原因
设计阶段不得进入具体实现细节。

6. **识别风险与不确定性**
设计中必须指出：
- 可能出错或存在不确定性的地方
- 尚未验证的假设
- 需要在实现早期优先确认的问题

7. **设计完成后才能进入实现**
在开始实现前，必须明确声明：
设计已完成，下面开始按设计进行实现。
若实现过程中发现设计不成立，必须回退并修订设计。

**Rule of thumb**
先设计，再编码；想清楚问题，代码自然会变得简单。
