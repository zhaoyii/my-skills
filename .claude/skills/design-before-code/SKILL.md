---
name: design-before-code
description: 在写代码之前，先完成程序设计，明确问题、目标以及环境约束。适用于任何软件工程任务。
---

在解决任何软件工程问题之前，始终包含:

1. **禁止直接写代码**：在设计完成之前，不得编写、修改或生成任何代码。
2. **明确问题、目标和环境约束**：弄明白核心问题是什么，为什么要解决这个问题？明确要实现的目标，以及相关环境约束。
3. **给出设计方案**：设计整体结构或模块划分，关键职责边界，主要数据流或控制流程。对重要的设计取舍，分析原因。
4. **设计方法**：采用用例分析（Use Case Analysis） 作为核心设计方法。识别用例（Use Cases），提取需求与设计依据。从用例或功能分析中，统计某个需求出现或触发的概率，作为设计决策的参考。
