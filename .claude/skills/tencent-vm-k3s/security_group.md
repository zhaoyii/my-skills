# 云安全组基础说明

## 一、安全组（Security Group）

安全组可以理解为云服务器上的"虚拟防火墙"，用于控制实例的网络访问策略。

它本质是一组绑定在云资源上的规则集合，用来决定：

-   哪些流量可以进入（入站）
-   哪些流量可以出去（出站）

示例（概念）：

``` text
安全组: web-sg
- 入站规则
  - 允许 SSH (22) 来自指定IP
  - 允许 HTTP (80) 来自所有IP
- 出站规则
  - 允许所有外部访问
```

------------------------------------------------------------------------

## 二、入站规则（Inbound / Ingress）

入站规则用于控制：**外部 IP 能否访问你的服务器的指定端口**。

### 示例 1：限制 SSH 登录

``` text
来源IP: 203.0.113.10/32
协议: TCP
端口: 22
```

含义： - 只有该 IP 可以通过 SSH 登录服务器

------------------------------------------------------------------------

### 示例 2：开放 Web 服务

``` text
来源IP: 0.0.0.0/0
协议: TCP
端口: 80
```

含义： - 任何人都可以访问你的 HTTP 服务

------------------------------------------------------------------------

### 示例 3：仅允许内网访问数据库

``` text
来源IP: 10.0.0.0/16
协议: TCP
端口: 3306
```

含义： - 仅内网机器可以访问 MySQL

------------------------------------------------------------------------

## 三、出站规则（Outbound / Egress）

出站规则用于控制：**你的服务器能访问哪些外部 IP 和端口**。

### 示例 1：允许所有出站（常见默认）

``` text
目标IP: 0.0.0.0/0
协议: ALL
端口: ALL
```

含义： - 服务器可以访问任意外部服务 - 用于： - 下载依赖 - API 调用 -
拉取镜像

------------------------------------------------------------------------

### 示例 2：限制访问特定服务

``` text
目标IP: 8.8.8.8/32
协议: UDP
端口: 53
```

含义： - 仅允许访问指定 DNS 服务器

------------------------------------------------------------------------

### 示例 3：限制访问内部服务

``` text
目标IP: 10.0.0.0/16
协议: TCP
端口: 3306
```

含义： - 仅允许访问内网数据库

------------------------------------------------------------------------

## 总结

-   安全组：云上的虚拟防火墙规则集合
-   入站规则：控制"谁可以访问我"
-   出站规则：控制"我可以访问谁"
